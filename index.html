Musync

GMM-based recommendation engine

A full-stack social music platform built around a Gaussian Mixture Model recommendation engine deployed as a FastAPI microservice on Render. The GMM clusters user listening profiles across five Spotify audio features and powers multiple recommendation strategies: multi-user probability overlap for group playlists (multiplying log-likelihoods across all members' GMMs), a tolerance-based scoring system for mixed groups, and collaborative filtering via weighted GMM centroid distance in 5D feature space to match similar users. The system also implements online learning — dynamically shifting cluster weights and means using predict_proba feedback on likes/skips, creating an adaptive taste profile that evolves with user behavior.

def compute_party_overlap(user_gmms, candidates, vibe):
    """Venn diagram overlap — multiply probabilities
    across all users' GMMs to find universal hits."""
    gmms = [rebuild_gmm(u) for u in user_gmms]
    X = extract_features(candidates)
    scores = []

    for i, feats in enumerate(X):
        # Log-likelihood from each user's GMM
        total_log_prob = sum(
            gmm.score_samples([feats])[0]
            for gmm in gmms
        )
        combined = np.exp(total_log_prob)

        # Apply contextual vibe penalty
        penalty = apply_vibe_penalty(candidates[i], vibe)
        scores.append({
            "track_id": candidates[i].id,
            "score": float(combined * penalty)
        })

    scores.sort(key=lambda x: x["score"], reverse=True)
    return scores
def get_gmm_centroid(params):
    """Weighted center of all clusters —
    the 'absolute average' of a user's taste."""
    weights = np.array(params["weights"])
    means = np.array(params["means"])
    return np.average(means, axis=0, weights=weights)

def compute_cf_distance(target_gmm, global_gmms):
    """Finds nearest users by comparing GMM centroids
    via Euclidean distance in 5D feature space."""
    target = get_gmm_centroid(target_gmm)
    results = []

    for uid, params in global_gmms.items():
        centroid = get_gmm_centroid(params)

        # Euclidean distance → similarity score
        dist = np.linalg.norm(target - centroid)
        similarity = 1.0 / (1.0 + float(dist))

        results.append({
            "user_id": uid,
            "similarity": similarity
        })

    results.sort(key=lambda x: x["similarity"],
                 reverse=True)
    return results
def apply_feedback(user_gmm, track_features, action):
    """Online learning — dynamically shifts GMM
    cluster weights/means on each like or skip."""
    gmm = rebuild_gmm(user_gmm)
    X = extract_features([track_features])

    # Which clusters does this track belong to?
    probs = gmm.predict_proba(X)[0]

    weights = np.array(user_gmm["weights"])
    means = np.array(user_gmm["means"])
    learning_rate = 0.1

    if action == "like":
        # Boost matching clusters, shift means closer
        weights += learning_rate * probs
        for i in range(len(means)):
            means[i] += learning_rate * probs[i] \
                        * (X[0] - means[i])
    elif action == "skip":
        # Penalize matching clusters (min 1%)
        weights -= learning_rate * probs
        weights = np.clip(weights, 0.01, None)

    # Re-normalize to sum to 1.0
    weights /= np.sum(weights)

    return {
        "weights": weights.tolist(),
        "means": means.tolist(),
        "covariances": user_gmm["covariances"]
    }

Intunate

Interactive guitar chord voicing explorer

Built as a deep-dive into domain modeling and algorithmic generation — encoding music theory (intervals, chord formulas, scale patterns) as composable data structures, then generating all valid fretboard voicings computationally. The engine maps 15 chord types across 8 tuning systems using modular arithmetic over the 12-tone chromatic space. Also an exercise in browser-native audio synthesis via the Web Audio API, building a real-time instrument from oscillators without external libraries.

Pickleball Point Detection

Computer vision pipeline for automated scoring

A computer vision proof-of-concept for automated pickleball/tennis point detection. Uses a fine-tuned ResNet50 to detect 14 court keypoints, a custom-trained YOLOv5 model for ball tracking, and a pre-trained YOLOv8 for player detection. The pipeline stitches these together to analyze ball trajectory, detect bounces via peak detection on the Y-coordinate time series, classify hits, and determine in/out calls using convex hull geometry over the detected court boundary.

def get_court_keypoints(frame, model, transform, device):
    h, w = frame.shape[:2]
    img_rgb = cv2.cvtColor(frame, cv2.COLOR_BGR2RGB)
    img_tensor = transform(img_rgb).unsqueeze(0).to(device)

    with torch.no_grad():
        outputs = model(img_tensor)

    kps = outputs[0].cpu().numpy()

    # Map keypoints back to the original frame dimensions
    kps[::2] *= w / 224.0
    kps[1::2] *= h / 224.0

    return [(int(kps[i]), int(kps[i+1]))
            for i in range(0, len(kps), 2)]
